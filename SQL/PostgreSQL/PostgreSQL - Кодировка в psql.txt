Словами не описать, какими эмоциями был сопряжён процесс. Потому не буду.

Сначала прочитайте Дополнение.

Windows 7 SP 1 x64
PostgreSQL 8.4.12 x32

Что из нижеперечисленного помогло - не знаю. Потому просто мои шаги:

1) заменить в реестре в ветке HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\Code Page с "1252"="c_1252.nls" на "1252"="c_1251.nls";
2) создать в pdAdmin3 новую БД, выбрав кодировку WIN1251 и в качестве шаблона template0;
3) добавить в эту новую БД вашу схему. Причём, у меня в скрипте создания схемы стояла команда SET client_encoding = 'UTF8';
4) открыть cmd.exe, Свойства - Шрифт - Lucida Console;
5) запустить psql -d ВАШАБАЗАКОТОРАЯТЕПЕРЬWIN1251 -U ВАШЛОГИН, там выполнить set client_encoding='win1251'.

После этих действий результаты запросов в psql нормально отображают русские поля.

Дополнение
----------

На другом компьютере проверял - хватило вот этих действий:
1) открыть cmd.exe, Свойства - Шрифт - Lucida Console, шрифт выбрал 14, но это просто чтобы покрупнее было;
2) выполнить команду chcp 1251. Ответ выведется иероглифами, но ответ будет положительный;
3) запустить psql -d ВАШАБАЗА -U ВАШЛОГИН, там выполнить set client_encoding='win1251'.

Как видите, создавать новую БД чисто на WIN1251 не надо.

Если не получится, то пробуйте весь комплекс мероприятий от начала документа.
